## 📖 主体讲稿

### Slide 1 — 先建立今天的直觉

把今天的内容想成一个工具箱。遇到题目时，先问三件事：它在解决什么问题？输入是什么？输出应该长什么样？只要这三件事想清楚，选择题、连线题和开放实战题都会变简单。

### Slide 2 — 考点 1-4：逐题白话拆解

- 当题目问「区别：Agent RAG 和基础 RAG 的最大区别是？」时，正确理解是：AI 可以自主判断是否需要检索、检索什么、结果够不够好——像一个会思考的研究员。
- 当题目问「场景：用户问「比较 A 产品和 B 产品的优缺点」，Agent RAG 会怎么做？」时，正确理解是：分别检索 A 产品和 B 产品的文档，再综合对比——自动拆分成两个子检索任务。
- 当题目问「概念：AI 自主判断下一步行动的能力叫做 ___ 能力。」时，正确理解是：Agent（智能体）。
- 当题目问「概念：GraphRAG 中的「Graph」指什么？」时，正确理解是：知识图谱——用节点和关系边来表示知识之间的连接。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 3 — 考点 5-8：逐题白话拆解

- 当题目问「优势：传统 RAG 很难回答但 GraphRAG 能回答的问题类型是？」时，正确理解是：多跳推理——需要跨多个实体追踪关系的问题。
- 连线题要记成配对表：基础 RAG = 检索→生成，固定流程；Agent RAG = AI 自主判断检索策略；GraphRAG = 基于知识图谱的多跳检索；Multi-query RAG = 一个问题拆成多角度检索。
- 开放实战要讲清楚：直接写 Prompt：围绕「张三任职年限和项目经历查询」，写一个可以直接复制给大模型使用的 Agent RAG 与 GraphRAG Prompt。要求包含：角色/目标、输入信息、执行步骤、输出格式、边界限制。 评分时看这些点：是否能直接复制使用；是否围绕 Agent RAG 与 GraphRAG 和「张三任职年限和项目经历查询」展开；是否包含角色/目标、输入、步骤、输出格式和边界；是否能降低「代词消解错误或检索不完整」风险；语言是否清晰不绕。
- 开放实战要讲清楚：结构化表达：为「张三任职年限和项目经历查询」写一份可直接复制使用的任务说明。必须面向「HR 知识库用户」，并明确输入、处理步骤、输出格式和禁止事项。 评分时看这些点：结构是否清晰；输入/步骤/输出是否完整；是否符合 HR 知识库用户 的理解水平；禁止事项是否能降低「代词消解错误或检索不完整」风险。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 4 — 考点 9-12：逐题白话拆解

- 开放实战要讲清楚：边界处理：列出「张三任职年限和项目经历查询」在真实使用中最容易失败的 4 种情况，并分别写出 AI 应该如何回应或兜底。 评分时看这些点：是否包含真实边界场景；每个兜底是否具体可执行；是否避免让 AI 编造；是否覆盖「代词消解错误或检索不完整」这个核心风险。
- 开放实战要讲清楚：输出格式设计：为「Agent RAG ReAct 流程」设计一个标准输出模板。要求模板字段清楚、顺序合理，并说明每个字段为什么需要。 评分时看这些点：模板是否能支撑 3 轮 Thought/Action/Observation + 综合答案；字段是否完整且不冗余；是否说明字段用途；是否便于后续复制、评审或程序解析。
- 开放实战要讲清楚：质量评审：假设另一个同学提交了「Agent RAG ReAct 流程」。请写一份评分标准，至少包含 5 个维度，并给出每个维度的满分表现和扣分点。 评分时看这些点：是否有 5 个以上评分维度；维度是否贴合 Agent RAG 与 GraphRAG；是否区分满分表现和扣分点；是否能指导同学改进。
- 开放实战要讲清楚：示例驱动：为「张三任职年限和项目经历查询」补充 2 组高质量示例：一组是简单输入，一组是困难输入。每组都要写出理想输出，并解释困难输入难在哪里。 评分时看这些点：是否有简单和困难两组示例；输入输出是否配套；困难点解释是否准确；示例是否能帮助大模型学会 Agent RAG 与 GraphRAG。

- 开放实战要讲清楚：约束优化：把「Agent RAG ReAct 流程」压缩成一个更稳的版本：要求减少废话，但不能丢掉角色、任务、边界、输出格式和检查标准。 评分时看这些点：是否保留关键约束；是否去掉空话；是否更容易执行；是否仍能防止「代词消解错误或检索不完整」。

### Slide 6 — 小白最容易混淆的点

- 如果题目问“概念”，先给一句人话定义，再补英文术语。
- 如果题目问“场景”，先判断任务类型，再选工具或 Prompt 策略。
- 如果题目问“Debug”，先找根因，不要直接堆更多功能。
- 如果题目问“Boss 综合”，答案要有结构：目标、步骤、格式、边界、检查。

### Slide 7 — 本课实战模板

今天的开放题是：直接写 Prompt：围绕「张三任职年限和项目经历查询」，写一个可以直接复制给大模型使用的 Agent RAG 与 GraphRAG Prompt。要求包含：角色/目标、输入信息、执行步骤、输出格式、边界限制。

可以按这个模板来答：

1. 先写清楚角色或目标。
2. 再写输入是什么、处理步骤是什么。
3. 明确输出格式，不要让 AI 自由发挥。
4. 加上边界条件：不知道怎么办、失败怎么办、哪些不能做。
5. 最后检查评分标准：结构完整、细节具体、能直接拿去用。
