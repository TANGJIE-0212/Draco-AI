# Week 2 Day 7：Prompt 工程综合实战

## 🎬 开场（10-20 秒）

今天我们讲 Prompt 工程 里的「Prompt 工程综合实战」。这一课不要死记硬背，先抓住一个小白也能懂的主线：第二周 Boss Battle：Prompt Engineering 综合大考。

这支视频必须覆盖当天所有题目，一共 10 个考点。看完以后，你不仅知道答案，还能说出为什么。

## 📖 主体讲稿

### Slide 1 — 先建立今天的直觉

把今天的内容想成一个工具箱。遇到题目时，先问三件事：它在解决什么问题？输入是什么？输出应该长什么样？只要这三件事想清楚，选择题、连线题和开放实战题都会变简单。

### Slide 2 — 考点 1-4：逐题白话拆解

- 当题目问「System Prompt：System Prompt 的四大要素是？」时，正确理解是：角色定义、能力边界、输出风格、禁忌事项。
- 当题目问「结构化：CRISPE 框架中的 'C' 代表什么？」时，正确理解是：Capacity 角色能力。
- 当题目问「Few-Shot：给 AI 提供 0 个示例直接让它执行任务，叫 ___-Shot。」时，正确理解是：Zero。
- 当题目问「CoT：「Let's think step by step」这句话的作用是？」时，正确理解是：激活 Zero-Shot Chain-of-Thought，让 AI 先输出推理过程再给答案。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 3 — 考点 5-8：逐题白话拆解

- 当题目问「输出控制：让 AI 输出 100% 合法 JSON 的最稳定方案是？」时，正确理解是：使用 Function Calling + JSON Schema 约束。
- 当题目问「安全：Prompt Injection 和 Jailbreak 的关键区别？」时，正确理解是：Injection 是覆盖系统指令（改行为），Jailbreak 是绕过安全限制（做禁止的事）。
- 开放实战要讲清楚：组合至少 2 种技巧：下面是一个简单 Prompt：

「请帮我把用户评论分类成正面/负面，并提取关键词。」

请从「CRISPE / Few-Shot / CoT / JSON / 防越狱」中至少挑 2 项加进去，把它升级成一个更稳的版本。 评分时看这些点：是否明确说明用了哪 2 种以上技巧；是否真的把这些技巧体现在 Prompt 内容里（不只是嘴上说）；改写后是否比原版更可控；是否能直接复制使用。
- 开放实战要讲清楚：写一个 Few-Shot + JSON 的 Prompt：请你写一个完整的「商品评论情感分类」Prompt，要求：
1) 至少 2 组示例；
2) 输出严格 JSON（包含 label 和 score 两个字段）；
3) 明确说明 label 的取值集合。 评分时看这些点：是否同时用到 Few-Shot 和 JSON；是否给了至少 2 组示例；JSON 字段是否清晰；是否能直接给 AI 用。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 4 — 考点 9-10：逐题白话拆解

- 开放实战要讲清楚：为法律咨询场景写防风险提示：场景：「面向普通用户的法律咨询 AI」。请你写出至少 2 条 AI 必须严格遵守的防风险提示，并解释为什么这条规则重要。 评分时看这些点：是否真的针对“普通用户法律咨询”场景；是否覆盖“不下定论 / 不替代律师 / 不编造法条”等核心风险；是否解释了为什么；语言是否清楚。
- 开放实战要讲清楚：综合大题：法律咨询 AI：请写一份完整的「法律咨询 AI」System Prompt，准备投入产品中。要求：
1) 角色与服务范围；
2) CRISPE 风格的角色 / 背景说明；
3) 至少 1 个 Few-Shot 问答示例（展示理想回答）；
4) 输出格式（结构化字段或 JSON 任选其一）；
5) 至少 3 条防风险规则（不下定论、不编法条、不替代律师等）；
6) 一段被诱导“给具体诉讼建议”时的拒绝话术。 评分时看这些点：是否覆盖 6 项；CRISPE/Few-Shot/输出控制/安全规则是否真的体现；示例和拒绝话术是否真实可用；是否避免空话；整体是否能直接上线；语言是否清晰。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 5 — 小白最容易混淆的点

- 如果题目问“概念”，先给一句人话定义，再补英文术语。
- 如果题目问“场景”，先判断任务类型，再选工具或 Prompt 策略。
- 如果题目问“Debug”，先找根因，不要直接堆更多功能。
- 如果题目问“Boss 综合”，答案要有结构：目标、步骤、格式、边界、检查。

### Slide 6 — 本课实战模板

今天的开放题是：组合至少 2 种技巧：下面是一个简单 Prompt：

「请帮我把用户评论分类成正面/负面，并提取关键词。」

请从「CRISPE / Few-Shot / CoT / JSON / 防越狱」中至少挑 2 项加进去，把它升级成一个更稳的版本。

可以按这个模板来答：

1. 先写清楚角色或目标。
2. 再写输入是什么、处理步骤是什么。
3. 明确输出格式，不要让 AI 自由发挥。
4. 加上边界条件：不知道怎么办、失败怎么办、哪些不能做。
5. 最后检查评分标准：结构完整、细节具体、能直接拿去用。
