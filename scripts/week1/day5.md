# Week 1 Day 5：记忆的瓶颈：Context Window (上下文窗口)

⏱️ 时长：约 7-9 分钟 | 适用：NotebookLM 视频生成 / PPT 讲解

---

## 🎬 开场（10-20 秒）

今天我们讲 AI 基础原理 里的「记忆的瓶颈：Context Window (上下文窗口)」。这一课不要死记硬背，先抓住一个小白也能懂的主线：第一章：AI 的短期记忆 (RAM)。

这支视频必须覆盖当天所有题目，一共 14 个考点。看完以后，你不仅知道答案，还能说出为什么。

## 📖 主体讲稿

### Slide 1 — 先建立今天的直觉

把今天的内容想成一个工具箱。遇到题目时，先问三件事：它在解决什么问题？输入是什么？输出应该长什么样？只要这三件事想清楚，选择题、连线题和开放实战题都会变简单。

### Slide 2 — 考点 1-4：逐题白话拆解

- 当题目问「概念辨析：关于“训练数据”和“Context”，以下说法正确的是？」时，正确理解是：Context 是临时的，一刷新网页或开启新对话，之前的 Context 就消失了。
- 当题目问「计算直觉：如果把 Context Window 从 4k 提升到 8k，理论上 Attention 矩阵的计算量会增加多少倍？」时，正确理解是：4倍 (平方级增长)。
- 当题目问「术语：当对话长度超过了模型限制，最早的信息会被强制丢弃，这种机制通常被称为 ___ (Sliding Window)。」时，正确理解是：滑动 (Sliding)。
- 当题目问「场景模拟：你给 AI 发了一篇 1 万字的论文（占满了窗口），要求它“详细扩写”这篇论文。结果 AI 写了一半就断了（Truncated）。原因是？」时，正确理解是：Input 占用了太多空间，剩余的 Context 不足以容纳完整的 Output。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 3 — 考点 5-8：逐题白话拆解

- 当题目问「Debug：用户反馈：“我和 AI 聊了 1 小时，它突然忘了我最开始设定的名字。” 最合理的解释是？」时，正确理解是：初始 Prompt 已经被挤出了滑动窗口的范围。
- 当题目问「策略：为了缓解“迷失在中间”现象，重要的指令（Instruction）最好放在 Prompt 的哪个位置？」时，正确理解是：最开头或最结尾。
- 连线题要记成配对表：Step 1 = 用户提问 (Query)；Step 2 = 在数据库中检索相关片段 (Retrieve)；Step 3 = 将片段和问题拼接成 Prompt (Augment)；Step 4 = AI 生成答案 (Generate)。
- 当题目问「选型：什么情况下 必须 使用 RAG，而不是仅靠模型自带的知识？」时，正确理解是：询问今天的公司内部库存数据（私有且实时变化）。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 4 — 考点 9-12：逐题白话拆解

- 当题目问「原理：RAG 技术本质上是用」时，正确理解是：搜索/检索。
- 当题目问「陷阱：有了 100万 Token 的超大窗口模型（如 Gemini 1.5 Pro），RAG 就会彻底消亡吗？」时，正确理解是：不会，RAG 在成本、速度和私有权限控制上仍有优势。
- 当题目问「计算：假设 1000 Token = $0.01。每次都把整本《红楼梦》（约100万 Token）丢给 AI 提问，和用 RAG 每次只检索 2000 Token 相比，成本差异大约是？」时，正确理解是：前者成本是后者的 500 倍。
- 当题目问「Debug：使用了 RAG 后，AI 回答：“根据提供的文档，我不知道答案。” 这说明？」时，正确理解是：这是正常的，说明检索到的文档里确实没有相关信息，AI 诚实地回答了。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 5 — 考点 13-14：逐题白话拆解

- 当题目问「幻觉：相比于直接问模型，使用 RAG 通常能 ___ 幻觉（Hallucination）。」时，正确理解是：减少 (因为有事实依据作为参考)。
- 连线题要记成配对表：Context Window = 昂贵的短期工作台 (RAM)；Sliding Window = 处理超长对话的妥协机制；RAG = 先搜索，再回答；Vector Database = RAG 的外挂图书馆；In-Context Learning = 利用上下文现学现卖。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 6 — 小白最容易混淆的点

- 如果题目问“概念”，先给一句人话定义，再补英文术语。
- 如果题目问“场景”，先判断任务类型，再选工具或 Prompt 策略。
- 如果题目问“Debug”，先找根因，不要直接堆更多功能。
- 如果题目问“Boss 综合”，答案要有结构：目标、步骤、格式、边界、检查。
