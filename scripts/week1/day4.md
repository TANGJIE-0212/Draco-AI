## 📖 主体讲稿

### Slide 1 — 先建立今天的直觉

把今天的内容想成一个工具箱。遇到题目时，先问三件事：它在解决什么问题？输入是什么？输出应该长什么样？只要这三件事想清楚，选择题、连线题和开放实战题都会变简单。

### Slide 2 — 考点 1-4：逐题白话拆解

- 当题目问「对比：相比于旧的 RNN 模型，Transformer 最大的优势是？」时，正确理解是：可以并行训练，速度极快且能捕捉长距离依赖。
- 当题目问「原理：如果去掉了位置编码，Transformer 处理 'Alice hit Bob' 和 'Bob hit Alice' 时，会认为它们...」时，正确理解是：完全相同。
- 当题目问「直觉：为什么位置编码是直接“加”在词向量上，而不是拼接在后面？」时，正确理解是：为了节省内存空间。
- 连线题要记成配对表：Query = 你在搜索框输入的关键词；Key = 网页的标题和元数据；Value = 网页的正文内容；Weight = 匹配的相关性分数。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 3 — 考点 5-8：逐题白话拆解

- 当题目问「直觉：在处理句子 'The cat sat on the mat' 时，当 Query 是 'sat' (坐) 时，它应该给哪个词的 Key 最高的关注度？」时，正确理解是：'cat' (因为要知道是谁坐)。
- 当题目问「流程：原始分数经过 Softmax 层后，所有分数的总和必然等于...」时，正确理解是：1。
- 当题目问「计算题：我是模型
假设 V(cat) 的数值是 10，V(mat) 的数值是 5。
经过 Softmax 计算出权重：cat (0.8), mat (0.2)。
最终输出的混合向量值是多少？」时，正确理解是：9.0 (0.8*10 + 0.2*5)。
- 当题目问「概念：经过 Self-Attention 层处理后，每个单词的向量发生了什么变化？」时，正确理解是：它融合了上下文其他词的信息（变成了 Contextual Embedding）。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 4 — 考点 9-12：逐题白话拆解

- 当题目问「推理：在句 A (tired) 中，Token 'it' 关注度最高的词应该是？」时，正确理解是：animal。
- 当题目问「推理：在句 B (wide) 中，Token 'it' 关注度最高的词应该是？」时，正确理解是：street。
- 当题目问「原理：Masked Attention 主要用于 Transformer 的哪个部分？」时，正确理解是：Decoder (解码器，如 GPT)。
- 当题目问「后果：如果没有 Mask，GPT 在训练时会发生什么？」时，正确理解是：它会直接“看到”答案，学不到任何预测能力。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 5 — 考点 13-16：逐题白话拆解

- 当题目问「类比：多头注意力就像...」时，正确理解是：你需要同时关注：剧情、演技、配乐、特效等多个维度。
- 当题目问「结构：Attention 层负责信息的」时，正确理解是：路由/聚合。
- 当题目问「Context Window：Context Window (上下文窗口) 的大小瓶颈主要来自？」时，正确理解是：Attention 矩阵计算的复杂度 (它是序列长度的平方级增长)。
- 连线题要记成配对表：1. 入口 = Input Embedding + Positional Encoding；2. 聚合 = Masked Multi-Head Attention；3. 消化 = Feed Forward Network (FFN)；4. 出口 = Linear + Softmax (预测概率)。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 6 — 小白最容易混淆的点

- 如果题目问“概念”，先给一句人话定义，再补英文术语。
- 如果题目问“场景”，先判断任务类型，再选工具或 Prompt 策略。
- 如果题目问“Debug”，先找根因，不要直接堆更多功能。
- 如果题目问“Boss 综合”，答案要有结构：目标、步骤、格式、边界、检查。
