## 📖 主体讲稿

### Slide 1 — 先建立今天的直觉

把今天的内容想成一个工具箱。遇到题目时，先问三件事：它在解决什么问题？输入是什么？输出应该长什么样？只要这三件事想清楚，选择题、连线题和开放实战题都会变简单。

### Slide 2 — 考点 1-4：逐题白话拆解

- 当题目问「概念：Greedy Search (贪婪搜索) 的主要缺点是什么？」时，正确理解是：生成的内容缺乏多样性，容易重复。
- 当题目问「原理：为了让 AI 像人一样说话，我们需要引入 ___ (Sampling)，即不一定选概率最高的词，而是按概率随机抽取。」时，正确理解是：采样 (Sampling)。
- 当题目问「参数直觉：当 Temperature 设置为 0 时，模型的行为最接近于？」时，正确理解是：Greedy Search (永远输出固定的唯一解)。
- 连线题要记成配对表：数学计算 / 代码生成 = 温度 0.0 - 0.2 (严谨)；创意写作 / 头脑风暴 = 温度 0.8 - 1.0 (发散)；数据抽取 / 格式转换 = 温度 0.1 (格式稳定)；日常闲聊 / 角色扮演 = 温度 0.6 - 0.8 (自然)。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 3 — 考点 5-8：逐题白话拆解

- 当题目问「陷阱：把 Temperature 调高到 1.5 甚至 2.0，最可能发生什么？」时，正确理解是：AI 开始胡言乱语，语法混乱，甚至生造单词。
- 当题目问「辨析：为什么 Top-P 通常比 Top-K 效果更好？」时，正确理解是：因为 Top-P 是动态调整候选数量的，能适应不同确信度的语境。
- 当题目问「实战：如果同时设置 Top-K=1 和 Temperature=1，实际起作用的是？」时，正确理解是：Top-K=1 (因为候选池被缩减到只剩 1 个，怎么随机都没用)。
- 当题目问「Debug：用户：“我用 API 写了一个自动回复邮件的机器人，但它有时候回复的内容太跳脱，甚至有点不礼貌。” 你的建议是？」时，正确理解是：调低 Temperature，并检查 Prompt 的语气约束。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 4 — 考点 9-12：逐题白话拆解

- 当题目问「Debug：用户：“我让 AI 生成 10 个不同的营销标题，结果它生成的 10 个看起来都差不多。” 你的建议是？」时，正确理解是：调高 Temperature 以增加多样性。
- 当题目问「技巧：在实际开发中，如果我们既想保持一定的逻辑性，又想有一点点变化，通常推荐的参数组合是：Temperature ≈ 0.7 且 Top-P ≈ ___ 。」时，正确理解是：0.9。
- 当题目问「种子：即使参数一样，为什么 OpenAI 有时还是会输出不同的结果？」时，正确理解是：因为 GPU 硬件层面的浮点数运算存在微小的非确定性。
- 连线题要记成配对表：Temperature = 概率分布的平滑度调节；Top-P = 动态截断尾部垃圾词；High Temp = 适合写诗、头脑风暴；Low Temp = 适合代码、数学、提取；Seed (种子) = 控制随机性的起始数。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 5 — 小白最容易混淆的点

- 如果题目问“概念”，先给一句人话定义，再补英文术语。
- 如果题目问“场景”，先判断任务类型，再选工具或 Prompt 策略。
- 如果题目问“Debug”，先找根因，不要直接堆更多功能。
- 如果题目问“Boss 综合”，答案要有结构：目标、步骤、格式、边界、检查。
