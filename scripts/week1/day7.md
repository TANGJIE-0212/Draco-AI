## 📖 主体讲稿

### Slide 1 — 先建立今天的直觉

把今天的内容想成一个工具箱。遇到题目时，先问三件事：它在解决什么问题？输入是什么？输出应该长什么样？只要这三件事想清楚，选择题、连线题和开放实战题都会变简单。

### Slide 2 — 考点 1-4：逐题白话拆解

- 连线题要记成配对表：瑞士银行：数据绝对不能出内网 = 本地部署开源模型 (Self-hosted)；独立开发者：快速验证创意，没钱买显卡 = 调用闭源 API (SaaS)；科研人员：需要研究模型底层架构 = 下载开源模型权重；企业客服：需要极高的稳定性且不想招运维 = 闭源 API。
- 当题目问「误区：开源模型意味着完全免费吗？」时，正确理解是：不是，虽然软件免费，但你需要支付昂贵的 GPU 推理成本（电费/算力租用）。
- 当题目问「应用：如果你的任务是“从一段文本中提取人名”，你会选择？」时，正确理解是：Llama-8B (小模型)。
- 当题目问「硬件：你想在自己的 24G 显存的显卡上跑一个 70B 的模型（通常需要 140G 显存），唯一可行的方法是？」时，正确理解是：使用极高程度的量化 (Quantization) 或 内存卸载 (Offloading)。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 3 — 考点 5-8：逐题白话拆解

- 当题目问「原理：AI 本质上是在计算什么？ P(wt | w0...wt-1)」时，正确理解是：给定前面的词，计算下一个词出现的条件概率。
- 当题目问「核心：Transformer 区别于 RNN 的最大优势是它可以 ___ (Parallel) 计算，并且捕捉长距离依赖。」时，正确理解是：并行 (Parallel)。
- 当题目问「Embedding：为什么 '国王 - 男人 + 女人' 能算出 '女王'？」时，正确理解是：因为在 Embedding 空间中，语义特征被编码成了向量方向，可以进行代数运算。
- 连线题要记成配对表：把字切成数 = Tokenizer；把数变成坐标 = Embedding；存储海量坐标 = Vector DB；计算关注点 = Attention Mechanism。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 4 — 考点 9-12：逐题白话拆解

- 当题目问「Context：当 Output Token 达到上限时，通常会发生什么？」时，正确理解是：句子会突然中断 (Truncated)。
- 当题目问「RAG：为什么有了 RAG 还需要 Prompt Engineering？」时，正确理解是：RAG 只是找资料，怎么利用这些资料生成好的回答，依然取决于 Prompt 的质量。
- 当题目问「Temperature：你在做一个自动写代码的插件，为了保证代码能运行（不瞎编函数名），Temperature 应设为？」时，正确理解是：0.0 (Greedy)。
- 当题目问「Prompt：Let's think step by step 是为了激活模型的什么能力？」时，正确理解是：Chain of Thought (CoT)。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 5 — 考点 13-15：逐题白话拆解

- 当题目问「Prompt：Few-Shot 是指在 Prompt 中提供」时，正确理解是：示例 (Examples)。
- 当题目问「终极应用：你要开发一个“法律文书助手”。
要求：严谨、引用法条准确、不用最新的网络新闻。
最佳技术栈组合是？」时，正确理解是：Temp=0.1 + RAG (挂载法条库) + System Role ('你是资深律师')。
- 连线题要记成配对表：算力层 = GPU / TPU；模型层 = Transformer / Weights；记忆层 = Vector DB / Context；应用层 = Prompt / Agent。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 6 — 小白最容易混淆的点

- 如果题目问“概念”，先给一句人话定义，再补英文术语。
- 如果题目问“场景”，先判断任务类型，再选工具或 Prompt 策略。
- 如果题目问“Debug”，先找根因，不要直接堆更多功能。
- 如果题目问“Boss 综合”，答案要有结构：目标、步骤、格式、边界、检查。
