## 📖 主体讲稿

### Slide 1 — 先建立今天的直觉

把今天的内容想成一个工具箱。遇到题目时，先问三件事：它在解决什么问题？输入是什么？输出应该长什么样？只要这三件事想清楚，选择题、连线题和开放实战题都会变简单。

### Slide 2 — 考点 1-4：逐题白话拆解

- 当题目问「定义：AI Agent 和普通 ChatGPT 的最大区别是？」时，正确理解是：Agent 不只是聊天——它能使用工具自主执行任务，而不仅仅是生成文字。
- 连线题要记成配对表：LLM（大脑） = 理解任务、推理决策；Tools（工具） = 搜索、计算、API 调用；Memory（记忆） = 记住之前的操作；Planning（规划） = 拆分任务、制定计划。
- 当题目问「公式：AI Agent = LLM + ___ + Memory + Planning。」时，正确理解是：Tools（工具）。
- 当题目问「概念：Function Calling 的作用是？」时，正确理解是：让 AI 能够自主判断何时需要调用外部工具（API/函数）来获取信息或执行操作。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 3 — 考点 5-8：逐题白话拆解

- 当题目问「流程：用户问「帮我订明天下午 3 点的会议室」，Agent 的执行流程是？」时，正确理解是：理解意图→调用日历工具查空闲→调用预订工具→确认结果→回复用户。
- 当题目问「框架：ReAct 的三个步骤是什么？」时，正确理解是：Thought（思考）→ Action（行动）→ Observation（观察），循环执行。
- 开放实战要讲清楚：直接写 Prompt：围绕「自动订机票 Agent」，写一个可以直接复制给大模型使用的 AI Agent Prompt。要求包含：角色/目标、输入信息、执行步骤、输出格式、边界限制。 评分时看这些点：是否能直接复制使用；是否围绕 AI Agent 和「自动订机票 Agent」展开；是否包含角色/目标、输入、步骤、输出格式和边界；是否能降低「未经确认直接支付下单」风险；语言是否清晰不绕。
- 开放实战要讲清楚：结构化表达：为「自动订机票 Agent」写一份可直接复制使用的任务说明。必须面向「差旅用户」，并明确输入、处理步骤、输出格式和禁止事项。 评分时看这些点：结构是否清晰；输入/步骤/输出是否完整；是否符合 差旅用户 的理解水平；禁止事项是否能降低「未经确认直接支付下单」风险。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 4 — 考点 9-12：逐题白话拆解

- 开放实战要讲清楚：边界处理：列出「自动订机票 Agent」在真实使用中最容易失败的 4 种情况，并分别写出 AI 应该如何回应或兜底。 评分时看这些点：是否包含真实边界场景；每个兜底是否具体可执行；是否避免让 AI 编造；是否覆盖「未经确认直接支付下单」这个核心风险。
- 开放实战要讲清楚：输出格式设计：为「工具设计和 ReAct 示例」设计一个标准输出模板。要求模板字段清楚、顺序合理，并说明每个字段为什么需要。 评分时看这些点：模板是否能支撑 Tools、3 轮 ReAct、人工确认点；字段是否完整且不冗余；是否说明字段用途；是否便于后续复制、评审或程序解析。
- 开放实战要讲清楚：质量评审：假设另一个同学提交了「工具设计和 ReAct 示例」。请写一份评分标准，至少包含 5 个维度，并给出每个维度的满分表现和扣分点。 评分时看这些点：是否有 5 个以上评分维度；维度是否贴合 AI Agent；是否区分满分表现和扣分点；是否能指导同学改进。
- 开放实战要讲清楚：示例驱动：为「自动订机票 Agent」补充 2 组高质量示例：一组是简单输入，一组是困难输入。每组都要写出理想输出，并解释困难输入难在哪里。 评分时看这些点：是否有简单和困难两组示例；输入输出是否配套；困难点解释是否准确；示例是否能帮助大模型学会 AI Agent。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 5 — 考点 13-15：逐题白话拆解

- 开放实战要讲清楚：约束优化：把「工具设计和 ReAct 示例」压缩成一个更稳的版本：要求减少废话，但不能丢掉角色、任务、边界、输出格式和检查标准。 评分时看这些点：是否保留关键约束；是否去掉空话；是否更容易执行；是否仍能防止「未经确认直接支付下单」。
- 开放实战要讲清楚：真实迁移：把今天的「AI Agent」迁移到你自己的一个工作/学习场景。请写出场景、目标用户、任务输入、期望输出和一版初始方案。 评分时看这些点：场景是否真实具体；是否能体现 AI Agent；输入输出是否明确；方案是否可执行；是否说明如何评估好坏。
- 开放实战要讲清楚：综合大题：完整产出一份「工具设计和 ReAct 示例」，用于「自动订机票 Agent」。必须包含：1）背景和目标；2）面向对象：差旅用户；3）完整方案正文；4）输出格式或交付物；5）至少 3 个质量检查点；6）针对「未经确认直接支付下单」的风险预案。要求写到可以直接拿去给 AI 或团队使用。 评分时看这些点：是否完整覆盖 6 个要求；是否紧扣 AI Agent 和 自动订机票 Agent；交付物是否具体、可执行、可复制；是否有明确质量检查；是否认真处理核心风险「未经确认直接支付下单」；是否避免空话和泛泛而谈。

讲的时候提醒学员：不要只背关键词，要把关键词放回真实场景里。题目一换皮，场景理解才是拿分关键。

### Slide 6 — 小白最容易混淆的点

- 如果题目问“概念”，先给一句人话定义，再补英文术语。
- 如果题目问“场景”，先判断任务类型，再选工具或 Prompt 策略。
- 如果题目问“Debug”，先找根因，不要直接堆更多功能。
- 如果题目问“Boss 综合”，答案要有结构：目标、步骤、格式、边界、检查。

### Slide 7 — 本课实战模板

今天的开放题是：直接写 Prompt：围绕「自动订机票 Agent」，写一个可以直接复制给大模型使用的 AI Agent Prompt。要求包含：角色/目标、输入信息、执行步骤、输出格式、边界限制。

可以按这个模板来答：

1. 先写清楚角色或目标。
2. 再写输入是什么、处理步骤是什么。
3. 明确输出格式，不要让 AI 自由发挥。
4. 加上边界条件：不知道怎么办、失败怎么办、哪些不能做。
5. 最后检查评分标准：结构完整、细节具体、能直接拿去用。
